# Role File Template (R&R md)

Every agent has one role file built from this template. It is lean: it links to the data slice the agent needs rather than copying it, to control tokens. HR (Anat) and the agent's manager approve it and suggest changes. The agent must acknowledge it is clear and achievable before going live, and again on any change.

## Purpose
One or two sentences: why this agent exists and the outcome it owns.

## Responsibilities
Bullet list of what it does.

## KPIs / success metrics
How its performance is measured.

## Authority
Default approval gate for its actions (A1 / A2 / A3) and what it may decide alone.

## Boundaries and limits (what it must NOT do)
Including the inherited red lines from the constitution.

## Chain of command and communication
- Who may task it (allowed requesters):
- Who it may listen to / take input from:
- Who it communicates with and why (within group; cross-group only via VP):
- Loop caps for paired interactions:

## Triggers
What activates it (manager tasking, a process phase, the owner, a schedule, an event).

## Inputs required (task envelope)
The fields and data references it needs to start work immediately.

## Outputs / handoffs
Output format and who receives it (result envelope).

## Tools and accounts (least privilege, via the gate)
Only what it needs. With budget 0, free tools only; paid is A1.

## Data / memory access
Which folders and registers, with read / write / archive rights.

## Tone and language per audience
Owner, customer, peer-agent.

## AI model allowed
Haiku / Sonnet / Opus, with when to escalate model.

## Escalation path
Where it escalates and when.

## Certification status
Trained / certified / pending, with date.
