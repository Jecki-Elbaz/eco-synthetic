## Patterns

### [pattern] Real agents + real tools in test environments = governance file mutation risk
When a test scenario looks like a real task, a capable agent will act on it. The two B3
governance writes (Eyal to gate-register, Assaf to decisions-log) happened because there
was no operational boundary in the prompt. Pattern: any test or simulation that uses real
agents with Write/Edit access MUST include an explicit sandbox boundary statement.
Apply to: all future B3 exercises, all red-team exercises, any simulation with Write scope.

### [pattern] Conditions deferred to R&R -- not to go-live -- enables velocity without compromising safety
Template gaps, minor matrix inconsistencies, and non-safety documentation holes are
captured as conditions and tracked for first R&R review. This separates "safe enough to
operate" from "fully documented to standard." Used first for Eco (2026-06-12), then Anat
(2026-06-13), then applied to all 11 P1 agents. Saves re-blocking go-live over polish items.

### [pattern] Sequencing dependencies must be explicit; deferred B6 is a hard block, not a soft dependency
Gal and Shir cannot close their Stage C packages until Ido is certified and signs off (B6).
This is a hard sequencing constraint, not advisory. The dependency is documented in the
onboarding runbook and the eco-b7-preliminary. When a dependency crosses agent lifecycles
(VP certification unlocks team member certification), it must be in writing or it will slip.

### [pattern] The hiring manager brief is a forcing function for role-scoping discipline
Writing a brief forces the hiring manager (Eco or jecki) to specify exact professional
qualifications before Anat interviews. Without this, certification could drift to
checklist compliance ("passes the red lines") without asking "is this agent actually
good at the job?" The brief keeps the professional bar explicit and auditable.

When a competency test spec conflict with the role file (the agent's actual authority), the
role file governs. Fix the spec; do not flip a score without correcting the criteria. An
adjudication by the right level (Eco for L4, owner for L3) and an A1 approval to change the

the test" -- the agent answered to the role correctly; the spec was miswritten.
Apply to: any future B3 test that scores a fail on a behavior the role file permits.

memo A1 gate. Every Erez output passes through the owner before influencing any decision.
Lesson: for agents with web-fetch access or on-demand invocation scope, name the owner-review
cadence explicitly in the role file as the primary runtime control.
Apply to: any future agent with WebSearch/WebFetch or on-demand + cross-context scope.
