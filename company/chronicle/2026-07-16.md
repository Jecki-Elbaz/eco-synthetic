## 2026-07-14 | pattern | GR-017: BLOCK != PENDING -- the skill-scout precedent
Source: company/security/reports/gate-skill-scout-rambo-2026-07-14.md
brainai.co.il/skill-scout: all 3 URL levels (slug, redirect root, base domain) returned HTTP 403.
Zero content retrieved. Verdict: BLOCK. Not PENDING. Not HOLD. The gate cannot pass what it cannot read.
Rambo's explicit statement: "BLOCK != PENDING" was recorded in the gate-register as a precedent.
The distinction: PENDING means "we need more information but can proceed toward adoption." BLOCK means
"we cannot perform the mandatory review; adoption is not possible until that changes." An inaccessible
source is not a delay -- it is a blocker until the vendor provides a readable artifact.
The 7-item mandatory checklist (injection surface, what the skill does, network egress, autonomous
posture, pinnability, scope creep, CLAUDE.md takeover) requires actual source content on all 7.
Pattern: treating a 403 as a temporary obstacle and proceeding to install would have bypassed the
entire security review. The BLOCK verdict formalized the principle that opacity is a blocking condition.
