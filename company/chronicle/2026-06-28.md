## Mistakes

### [mistake] 2026-06-16 -- Ido double-certified by two concurrent sessions
Two parallel Claude Code sessions each ran the full Ido certification pipeline independently
and produced correct but duplicated results. Root cause: no cross-session lock; each session
read master at start and did not know the other was doing the same work.
Lesson: until file-locking (T-0002) is built, agent buildout must be serialized through ONE
session, or coordinated via board check-in before starting.
Source: decisions-log 2026-06-16 Merge reconciliation: Ido double-certification incident.

### [mistake] 2026-06-15 -- Cloud session falsely accused Eco of fabricating work
A cloud session read master, found no Eco work products, and stated Eco had confabulated.
In fact, Eco had done real work on a LOCAL clone that was never pushed. The cloud session
failed its own verify-before-claim rule: checked one clone, stated a conclusion about both.
Lesson: not-in-git does not mean never-happened. State the scope of what you checked.
LOCAL SYNC RULE added to Eco.md: always pull from master before any status report.
Source: decisions-log 2026-06-15 Eco confabulation flags RETRACTED.

### [mistake] 2026-06-17 -- B3 candidates wrote to real governance files during tests
Two B3 candidates (Eyal, Assaf) appended to the live decisions-log and gate-register from
fabricated test inputs because scenarios were framed as real tasks, not tests. Both entries
had to be reverted.
Fix: B3 prompts now state explicitly it is a competency test and the agent must NOT write

Source: decisions-log 2026-06-17 Correction: removed a competency-test artifact.

### [mistake] 2026-06-22 -- 11-day bridge outage via silent auth failure
The bridge was healthy (--version returned 0, startup OK) but every message failed silently.
Root cause: CLAUDE_CODE_OAUTH_TOKEN unset; claude subprocess exited 1 with blank stderr;
bridge logged only stderr[:200] (blank). A --version check does NOT verify real auth.
Fix: on non-zero exit log stdout too; add a real --print startup auth probe; refuse to boot
if auth fails. A commit to the bridge does not fix a running process with a dead token.
Source: decisions-log 2026-06-22 Telegram bridge delivery outage ROOT CAUSE FOUND.

## Patterns

### [pattern] Behavioral layer precedes enforcement layer
Across 30 agents and a Phase 1 security audit: agent BEHAVIOR is strong from day one because
it is baked into each role file, tested at B3, reinforced by the soul Core Block. The
ENFORCEMENT layer (guard.py, file-locking, write-path restrictions) lags and is built
iteratively. Build governance into the agent mind first; enforcement catches edge cases.

### [pattern] New agent types not spawnable until session reload
B3 (behavioral test using fresh sub-agent spawn) cannot run in the same session that created
the agent type. The type is not in the registry until the next reload. Hard constraint: build
roles in session N, test in session N+1 after reload.

### [pattern] Show draft before approval -- owner will not A1 unseen changes
Established after Anat conditional cert was executed without prior owner review. Rule: always
show the exact draft content before requesting any A1. Applies to role files, policy changes,
go-lives, any A1-gated action.
Source: decisions-log 2026-06-13 Anat certification finalized process note.

### [pattern] Concurrent sessions drift; serialize or coordinate via the board
Twice in the build: parallel Claude Code sessions did the same work independently (Ido
double-cert; Shelly separation). Each session reads master at start and does not know what
other sessions are doing. Safe practice until file-locking is in place: one session owns one
domain at a time; the board is the coordination point before starting.
