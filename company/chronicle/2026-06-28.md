### [mistake] 2026-06-22 -- 11-day bridge outage via silent auth failure
The bridge was healthy (--version returned 0, startup OK) but every message failed silently.

if auth fails. A commit to the bridge does not fix a running process with a dead token.

## Patterns

### [pattern] Behavioral layer precedes enforcement layer
Across 30 agents and a Phase 1 security audit: agent BEHAVIOR is strong from day one because
it is baked into each role file, tested at B3, reinforced by the soul Core Block. The
ENFORCEMENT layer (guard.py, file-locking, write-path restrictions) lags and is built
iteratively. Build governance into the agent mind first; enforcement catches edge cases.

### [pattern] New agent types not spawnable until session reload
B3 (behavioral test using fresh sub-agent spawn) cannot run in the same session that created
the agent type. The type is not in the registry until the next reload. Hard constraint: build
roles in session N, test in session N+1 after reload.

### [pattern] Show draft before approval -- owner will not A1 unseen changes
Established after Anat conditional cert was executed without prior owner review. Rule: always
show the exact draft content before requesting any A1. Applies to role files, policy changes,
go-lives, any A1-gated action.
Source: decisions-log 2026-06-13 Anat certification finalized process note.

### [pattern] Concurrent sessions drift; serialize or coordinate via the board
Twice in the build: parallel Claude Code sessions did the same work independently (Ido
double-cert; Shelly separation). Each session reads master at start and does not know what
other sessions are doing. Safe practice until file-locking is in place: one session owns one
domain at a time; the board is the coordination point before starting.

## Patterns

### [pattern] Batch A1 go-lives: certify groups, not one-by-one
Both the P1 buildout (10 agents, 2026-06-17) and the P2 wave (10 agents, 2026-06-18) used
a batch A1 approval model: all agents in a wave were built, tested, and presented together,
then owner A1 covered the full batch. Faster than sequential approvals. Requires that every
agent in the batch is genuinely ready before the A1 request -- a weak member blocks the group.

### [pattern] Cross-project services always route back to eco-synthetic
Even after Shelly's standalone separation, her Security gate, Legal review, and HR cert run
through eco-synthetic agents (Rambo, Eyal, Anat). The capability lives in one place; the
client project calls back. This is the standing pattern for any future project separation.

### [pattern] Compliance requires two independent legs (Legal + Finance)
T-0005 revealed that compliance items look done when the legal review is complete, but the
finance implications (registration, invoicing, cost accrual) are a separate body of work.
Both legs must be formally signed off before a compliance item is truly closed.
